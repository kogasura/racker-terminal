# Racker Terminal

Windows 専用の自作ターミナルアプリ。内部（端末描画・PTY）は成熟ライブラリを使い、UI だけを自分好みに作り込む方針。

縦サイドバーでタブをグループ管理し、よく使うシェル構成を「お気に入り」としてワンクリック起動できます。

## インストール

リリースページから `Racker Terminal_<version>_x64-setup.exe` (NSIS) をダウンロードし、ダブルクリックでインストール → スタートメニューから起動。

### 必要環境

- Windows 10 (build 1809) 以降 / Windows 11
- WebView2 ランタイム（Windows 11 はプリインストール、Windows 10 は自動ダウンロード）
- フォント: **MonaspiceNe NF はバンドル済**（Cascadia Code / Consolas にフォールバック可）

### コード署名

v1.x では未署名のため、Windows Defender SmartScreen の警告が出る場合があります。
警告画面で「詳細情報」→「実行」で起動できます。

### アップデート

**v1.1 以降は起動時・起動中（1 時間ごと）に自動で更新を確認します。** 新しいバージョンがある場合、タイトルバーに更新ボタンが表示されます。クリックするとリリースノートを確認した上で、承認制でダウンロード・インストール・再起動が行われます。詳細は [docs/auto-update.md](docs/auto-update.md) を参照。

### v1.0.x で MSI 版を使っていた方へ

v1.1 から MSI インストーラーは廃止し、NSIS インストーラー (`setup.exe`) のみの配布になりました。`setup.exe` を実行すると新しい場所に v1.1 がインストールされます（旧 MSI 版とは別の場所）。旧 MSI 版が不要なら、Windows の「設定 → アプリ → インストールされているアプリ」から旧バージョン (1.0.x) をアンインストールしてください。

## 使い方

### お気に入り

よく使うシェル構成（シェル種別・起動ディレクトリ・引数・環境変数）を保存し、ワンクリックで新しいタブとして起動できる機能です。左サイドバー下部の **Favorites** セクションに一覧表示されます。

**登録する方法は 2 通り:**

1. **既存タブから登録（かんたん）** — タブを右クリック →「お気に入りに追加」。今開いているタブのシェル・ディレクトリ・引数・環境変数がそのまま引き継がれます。
2. **新規登録** — Favorites セクションの **`+ Add Favorite`** ボタンから、設定ダイアログを開いて手動入力します。

**登録後の操作（お気に入りを右クリック）:**

- **ここから spawn** — 設定内容で新しいタブを起動（クリックでも起動します）
- **編集** — 設定を変更
- **既定として設定 / 既定を解除** — 既定にすると ⭐ アイコンになり、`Ctrl+T`（既定タブを開く）で起動するシェルになります
- **削除**

お気に入りは D&D で並び替えでき、`Ctrl+Shift+1〜9` で上から 1〜9 番目をすばやく起動できます。

**設定ダイアログの項目:**

| 項目 | 必須 | 内容 |
|---|---|---|
| テンプレート | 任意 | 選択すると `タイトル`・`Shell`（一部は引数）を自動入力。WSL ディストロは自動検出され候補に並びます。あくまで雛形で、後から上書き編集できます |
| タイトル | **必須** | お気に入りの表示名 |
| Shell | 任意 | 例: `wsl.exe` / `pwsh.exe` / `cmd.exe`。**空欄なら既定シェル（nushell）** |
| CWD | 任意 | 起動ディレクトリ。例: `C:\Users\foo\projects`。**空欄ならホーム** |
| 引数 | 任意 | シェル起動時に渡すコマンドライン引数 → [書き方](#引数の書き方) |
| 環境変数 | 任意 | `KEY=VALUE` 形式の環境変数 → [書き方](#環境変数の書き方) |
| タブ名のデフォルト | 任意 | お気に入りタイトルと別のタブ名にしたい場合に指定。空欄ならタイトルを流用 |

### 引数の書き方

「引数」はシェルを起動するときに渡すコマンドライン引数です。**1 行に 1 つずつ**書くのがルールです。

ふだんターミナルでスペース区切りに打つ引数も、この欄では**改行して 1 つずつ分けて**書きます。

**例) WSL をホームディレクトリで起動したい**

ターミナルで打つコマンドはこうですが：

```
wsl.exe --cd ~
```

Racker Terminal のお気に入りでは、`Shell` 欄に `wsl.exe`、`引数` 欄に次のように **2 行**で書きます：

```
--cd
~
```

| 書き方 | 結果 |
|---|---|
| ✅ `--cd` と `~` を**別々の行** | `--cd` と `~` の 2 つの引数として正しく渡る |
| ❌ `--cd ~` を**1 行にまとめる** | 全体が 1 つの引数 `"--cd ~"` とみなされ、正しく動かない |

> 空行や前後の空白だけの行は無視されます。

### 環境変数の書き方

環境変数は **1 行に 1 つ、`KEY=VALUE` 形式**で書きます。

```
NODE_ENV=development
PATH=C:\custom\bin;%PATH%
```

- `KEY` は `A〜Z`・`a〜z`・`_` で始まり、英数字と `_` のみ使えます（POSIX 慣例）。不正な行があると保存時にエラー表示されます。
- ⚠️ env は**平文で保存**されます。API キー・パスワード等の機密値は入れないでください（[機密値の取り扱い注意](#機密値の取り扱い注意)）。

## キーボードショートカット

| ショートカット | 動作 |
|---|---|
| `Ctrl+T` | 既定タブ（既定のお気に入り）を開く |
| `Ctrl+Shift+T` | 閉じたタブを復元（最大 10 個まで履歴保持、再起動でクリア） |
| `Ctrl+Shift+W` | アクティブタブを閉じる |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | 次 / 前のタブへ移動 |
| `Ctrl+Shift+1〜9` | お気に入りの 1〜9 番目を開く |
| `Ctrl+C` | 選択範囲をコピー（選択なしのときは従来通り SIGINT） |
| `Ctrl+V` | クリップボードから貼り付け |
| `Ctrl+Enter` | 改行を挿入（ESC+CR を送出。Mac Terminal の `Option+Enter` 相当で、Claude Code 等の readline CLI が改行として認識する） |
| `Ctrl+クリック` | ターミナル上の URL を既定ブラウザで開く（`http:` / `https:` のみ。`javascript:` / `file:` / `data:` 等は無視） |

ターミナルへのファイルのドラッグ＆ドロップで、ファイルパスを貼り付けできます。

## 永続化される情報

アプリ再起動後も、タブ・グループ・お気に入りの構成が復元されます（localStorage に保存）。

| 情報 | 永続化 | 備考 |
|---|---|---|
| タブ・グループ・お気に入りの構成 | ✅ | |
| shell / cwd / 環境変数 | ✅ | 起動時に再 spawn |
| ユーザー編集したタブ名 (userTitle) | ✅ | OSC タイトルより優先 |
| Settings（フォント等） | ✅ | サイドバーフッタの ⚙ から変更 |
| scrollback（PTY 出力履歴） | ❌ | PTY と一蓮托生のため復元不可 |
| 実行中状態（active タブ・編集中） | ❌ | ランタイム情報 |
| shell 側の OSC タイトル | ❌ | 起動後に shell が再送信 |

## 機密値の取り扱い注意

お気に入りの `env`（環境変数）は **localStorage に平文 JSON で保存**されます。WebView2 の DevTools や `%LOCALAPPDATA%\<app>\EBWebView\` の LevelDB から読み取れる可能性があります。

**API キー・パスワード・個人トークン等の機密値は env に入れないでください。** 将来（Phase 5+）で OS keyring（Windows Credential Manager 等）への退避を検討中です。

## 技術スタック

| 層 | 採用 |
|---|---|
| 殻 | Tauri 2（WebView2 + Rust） |
| UI | React 19 + TypeScript + Vite + Tailwind CSS v4 |
| 端末描画 | [@xterm/xterm](https://www.npmjs.com/package/@xterm/xterm)（WebGL renderer + Canvas フォールバック） |
| PTY | Rust [portable-pty](https://crates.io/crates/portable-pty)（ConPTY 経由） |
| IPC | Tauri v2 Channel API |

## 開発

前提: Windows 11、Node.js 20+、Rust stable、nushell (`nu`) が PATH にあること。

```sh
npm install
npm run tauri dev
```

## 制約 / 方針

- Windows 専用、クロスプラットフォーム非対応
- ユーザー設定項目は最小限（フォント・配色・既定シェルは「押し付けるデフォルト」）
- ペイン分割なし、シングルウィンドウ

## バグ報告

問題があれば [GitHub Issues](https://github.com/kogasura/racker-terminal/issues) に報告してください。再現手順・OS バージョン (`winver`)・shell 種別（nushell / pwsh / wsl 等）を含めると助かります。
